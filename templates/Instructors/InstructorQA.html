Instructor Q&A

  • question_answerHow Can I Create a Course?
    In OneUp only the administrator can create courses and assign instructors to the courses. If you wish to have a course, please contact the OneUp administrator.
  • question_answerWhat’s a Course Description and How Do I Make One?
    The Course Description is a brief explanation of the content and activities of a course. To create one, in the Course Information menu select Course Description. Simply enter the description that is appropriate for the course and click Save.
  • question_answerWhat are Course Topics?
    Course Topics are the topics that are covered in the course. You can think of them as of chapters in a course textbook. Course topics are used to classify the warm-up challenges, so the students can find easily practice quizzes for the topic they wish to practice. To create a new Topic, in the Course Information menu select Course Topics. Click on Add New Topic and enter a name and, optionally, a position. The topics appear on the Topics list, sorted by the specified positions. When creating a warm-up challenge later, you can specify the course topic to which it belongs.
  • question_answerWhat are Activity Categories?
    Activity Categories play a role similar to Course Topics but are specifically for course activities (such as labs, assignments, etc., which are not automatically graded by the system). Activity Categories are used to classify course activities, so the students can find easily the activity they are looking for. To create one, in the Course Information menu select Activity Categories. Click on Add New Category and simply enter a name (i.e. Lab, Homework, etc.). After clicking Submit, the new category will appear in the Activity Categories List. When creating a new course activity later, you can assign it to one of the created activity categories.
  • question_answerWhat are Targeted Skills?
    Targeted Skills are exactly skills that are targeted in the specific course. You can use the learning objectives in your course as targeted skills. Skills are linked to problems (at the time of creation of the problems), as students learn skills as they complete problems. To create a Skill, in the Course Information menu select Targeted Skills. Click on Add New Skill and enter a name. After clicking Create, the new skill will be available to be assigned to problems.
  • question_answerWhat are Course Milestones?
    Course Milestones describe all of the required work in the class and how much (course points) each category is worth in total. The total of all milestones shall be equal to the maximum earnable amount of course points in the course (e.g. 100, or 1000, …). The course milestones are used in the creation of the Progress Bar, which is displayed on the student learning dashboard. To create a new Course Milestone, in the Course Information menu select Course Milestones. Click on Add New Milestone and enter the name, description, and the maximum total of points for your Milestone. After clicking Create, the new Milestone will appear in the list. If you don’t define Course Milestones, you have to disable the Progress Bar, which is shown in the Students/Achievements page. You can do that by going to the Course Information menu and select Course Configuration. Then disable the Progress Bar by turning the switch off.
  • question_answerHow are Students Added to Courses?

    Students do not create their own accounts, Instructors must make it for them. To do so, in the Course Information menu select Students. On this page, Instructors can view their students’ names, email, and the time of their most recent online activity. To add a new student, click on the Add New Student button to be taken to the Add New Student page. Simply fill in the student’s name, email, User Id, and create a password for them. Instructors are responsible for providing students their OneUp passwords. Click Submit to add the student to the course.

    Alternatively, you can import a file with students' registration information. You should be sure to select the format for the imported file and to enter the email domain for your university (the part after the @ sign) . Currently there are two supported file formats:

    • * CANVAS format: A file, exported from the CANVAS LMS: simply go to CANVA Grades, export (in the standard CSV format) and save the file on your computer. Then in OneUP's Create Student page choose the option "Import from CANVAS", and upload the previously exported file.
    • * OneUp format: A file in CSV (Comma Separated Values) format: you can prepare such file in any spreadsheet program (e.g. Excel) and save it in the CSV format. The spreadsheet has to have 3 columns: First name, Last name, and the local part (the part before the @ sign) of the student university email. Instructors should enter the domain for the email (the part after the @ sign) in the provided field. In this case you should choose the OneUp format for the file.

    If you wish to add students that are already registered in other OneUp courses, then click on the Add Existing Student button to be taken to the Add Students page. Simply select which students you wish to add to your course and click the Add to Class button.

  • question_answerWhat are the Course Configuration Options?
    In the Course Information menu, select Course Configuration to view the variety of options available for configuring the course. Course Availability makes the course available to students and is enabled by default. The Set Duration option is used to set the start and end date of the course. Once the Course End Date has passed, the course will no longer be available to students, even if Course Availability is enabled. The option Display Serious Challenges by Topics simply groups the challenges by Topics instead of the order in which they were created. The Progress Bar is used to track students progress in the course that works in tandem with Course Milestones. If no Course Milestones are defined, then the Progress Bar has no purpose, thus the option for it shall be disabled. Course Gamification is a completely optional feature to instructors that is disabled by default. Once enabled, a Game Features drop-down menu will appear in the instructor’s main menu which will allow you to configure the desired gamification features.
  • question_answerWhat are Attendance Streaks?
    A streak is a concept for informing users (through a number or other indicator) how many consecutive days they have completed a targeted action. OneUp supports streaks and the instructor can define class attendance streaks by setting what are the course 'consecutive' dates when the students have to be in attendance. To set an Attendance Streak, go to the Course Information menu and select Attendance Streaks. Note that this option is displayed only if you have enabled Streak Configuration in the Course Gamification Configuration page (under the Game Features menu). On the Student Attendance Streak Configuration page first select the days of the class and click the Submit button under the list of days. On the displayed calendar, you will see class dates added to the attendance streak. You can further adjust the streak dates by un-selecting specific dates during the semester on which there will be no classes. The calendar will update with the days you selected/unselected and will now have a streak label. Now click the Submit button under the calendar to create the streak.
  • question_answerWhat’s the Difference Between Warm-Up and Serious Challenges?
    In OneUp, course tests and quizzes are 'challenges' and the questions included in them are 'problems'. Challenges are divided into two types. Warm-up Challenges are primarily for student practice and do not directly contribute to the course grade. Typically the students can take a warm-up challenge as many times as needed to get a correct answer. The scores of the Warm-Ups are not shown in the Gradebook. By contrast, Serious Challenges are typically scheduled graded course exams or quizzes (as included in course syllabus). The primary idea being that the warm-up challenges should prepare the students for the serious challenges. Both types of challenges are automatically checked and the student is shown a score which reflects their performance.
  • question_answerHow do I Create Challenges?

    In the Course Content menu, select either Warm-Up or Serious Challenges. Click the Add New Challenge button to be taken to the Create Challenge page. After creating a name for the challenge, there are a plethora of configuration options available to Instructors. The Difficulty option has three options: Easy, Medium, and Hard. While selecting a difficulty is optional, it will categorize the challenges created. The option to Randomize Problems will make problems appear in a random order when taken by students instead of the order that they are added to the challenge.

    Instructors can set Availability constraints including visibility, attempts, time limit, and password-lock. By default the challenge will be visible to students, but the date range of that visibility and the due date for the challenge can be set. If a challenge is past its due date but still within the visible date range, it will still be available to students. Also by default, the number of attempts and the time limit are unlimited, but can be restricted. It should be noted that the time limit is in minutes, not seconds.

    Instructors can password lock a challenge. Simply create and confirm a password and distribute it to students as deemed necessary.

    The Curve is the amount of points that will be added automatically to each student's' score after they take the challenge. The option 'Possible points for a manually graded section of the challenge' allows the instructor to create a serious challenge with two parts: one consisting of OneUp problems which will be automatically scored and another of questions given (e.g. on paper) to the students, which the instructor will grade. After the grading, the instructor has to enter for each student the points scored on the manually graded part in the corresponding field in the Adjustment page. To go to this page, click on Adjustment in the menu besides the corresponding challenge (on the page displaying the list of challenges).

    The Topics drop-down menu displays the course topics previously created by the instructor. This option allows connecting the challenge to a particular course topic. If no topic is selected for the challenge, it will default to Unspecified, which will automatically place it in the Miscellaneous section. It is highly recommended that Topics are assigned to the warm-up challenges for ease of findability.

    Instructors also have the option to attach their own Tags to challenges. Attaching Tags allows challenges to be searched for by those Tags. The Feedback Options let Instructors determine how much feedback students get for each challenge. By default, the Display Correct Answers option is enabled, which shows the students the correct answers after they complete the challenge. Both the Correct and Incorrect Answer Feedback are disabled, for the feedback has to be specified for each problem in the challenge. After clicking Save, a new, empty challenge will be created. You then need to create problems for this challenge.
  • question_answerWhat is an Activity?
    An Activity is an assignment given to the students, which will not be automatically scored by the system, but will be graded by the instructor. The instructor can upload files for the students to complete assignments and students can submit files for the instructor to grade. Instructors can upload as many files as necessary, set how many upload attempts students are allowed, and set what category the Activity belongs to (i.e. Labs, Homework, etc.).
  • question_answerHow do I Create an Activity?

    In the Course Content menu select Activities, then click the Add New Activity button to be taken to the Create Activity page. After setting the Activity name, instructors can add a description that specifies the instructions for the Activity.

    The Default Points are the amount of points possible for the student to obtain. The Display Date range and Due Date specify how long the activity will be available and when is the submission deadline for the activity, respectively. The Activity Category selection option allows assigning the activity a specific category. Activity Categories can be defined by following the steps outlined in What are Activity Categories (under Course Information). If no category is selected, it will default to Uncategorized. The instructor has the option to decide if the Activity is graded and to allow students to upload files. The Activity Graded option simply allows the Activity to be graded by the Instructor. Graded activities are shown in the GradeBook.

    Enabling students to upload files means that instructors will be able to download student file submissions and grade them accordingly. Note that if the option to allow student uploads is not checked, the students will only be able to submit comments! Instructors can also restrict the amount of Upload Attempts, restricting to a certain amount of submission attempts. Instructors can upload their own files for the student to download to complete the Activity. If they wish to add more than one file, they can simply click the Add Another File to select another file to upload. After clicking Save, the new Activity will appear on the Activities list.
  • question_answerWhat are Problems and the Problem Pool?

    OneUp problems are questions or problems with answers that can be automatically checked for correctness. OneUp supports a plethora of problem types: Multiple Choice, True/False, Multiple Answers, Matching, Parson's Problems, and Dynamic problems. Problems can be assigned Targeted Skills, Difficulty, and Feedback to students.

    Problems are designed to be included in Challenges. They can be created either directly in a specific Challenge, or on the Problems page. In the latter case, they are stored in the OneUp Problem Pool (for later use) and are not assigned to any Challenge. The Problems page displays all such 'unassigned' problems.

    The problems in the Problem Pool are not ready for use just yet. A problem from the pool must be assigned to a Challenge first, then students will be able to access it. Once the problem is assigned, it is removed from the Problem pool and can be accessed via the Challenge. Problems can only be assigned points when they are added to a particular Challenge. If no points are assigned at that stage, the system will give zero points to the student for this problem.

    When you delete a problem from a Challenge, it is not removed from the database, but returns to the Problem Pool ready for reassignment. If you wish to truly Delete a problem, it must be deleted from the Problem Pool.

  • question_answerHow do I create a Problem?

    In the Course Content menu, select Problems. Here you will see the pool of already created problems, which are not assigned to any challenge. Click the Create Problem and select which type of problem you wish to create. The required information varies for every problem type, but most of the parameters are universal. The Problem Preview is the name the Instructor sets for the problem that is not seen by the student but is shown in the list of problems. The Problem Text contains either the the problem itself or instructions for the problem and is shown to the students. The Difficulty option has three options: Easy, Medium, and Hard. Like with challenges, selecting a difficulty is optional but helps categorize problems. For Parson problems, increasing the difficulty increases the amount of distractors shown to students (see Problems/How to Create a Parson's Problem).

    If Gamification is enabled, Skills can be added to problems along with the amount of Skill Points earned for the Skills. The skill points indicate how the problem contributes to increasing the level of student mastery of related skills. Instructors can choose to write feedback for when students answer the problem correctly and incorrectly. They can also write notes for themselves in the Instructor’s Notes. The author of the problem, by default, is the Instructor who creates the problem, but this can be changed if desired. In particular, if the instructor enters somebody else's problem, here they can acknowledge the authorship.

    Tags can be attached to problems. Attaching Tags further categorizes the problem and can make searching Problems simpler.

    After clicking Save, the new Problem will be added to the Problem Pool, ready to be included in a Challenge. It should be noted that points are not assigned to a Problem when created in the Problems section. Such points are assigned when the Problem is added to a particular Challenge.

  • question_answerHow Can I Add Problems to a Challenge?

    In the Course Content menu, select either Warm-Up or Serious Challenges. On the Challenge page, click on the menu (three vertical dots) beside the challenge to which you want to add problems. Select Problems. On the Problems for Challenge page, you can either click on the Create Problem button to create a new problem which will be automatically added to this Challenge, or click on the Search Problems button.

    If you click on the Search Problems button, this will bring you to the Search Problems page. Here you can select different filters for searching problems from the Problem Pool (Unassigned Problems) or from any other Challenge that you have created: by course topic, by problem type, by level of difficulty, by related skills, or by tags. Select the desired filter and click on Search.

    On the search result page you will see a list of all problems satisfying the search criteria. Select the problems that you wish to add to your challenge. For each selected problem enter the points for this problems in the challenge. Before selecting a problem, you may see it by clicking on the View button. When ready, click on the Add To Challenge button and the selected problems will be added to the challenge.

    Note that if you select problems from the Problem Pool, they will be removed from the pool after being added to the challenge. However, if you select problems from another challenge, they will not be removed from that challenge. Instead, copies of the problems will be created in the new challenge.

  • question_answerHow to Create a Matching Problem?

    Matching problems require students to match questions with their corresponding answers. To create one, go to the Course Content menu and select Problems. Click Create Problem and select Matching. With the exception of the Problem Answer fields, follow the guidelines in How do I create a Problem to create the problem. It is important to note that when you type a Matching Answer for Question X, that it only matches with that Question X. There must be a minimum of two Problem Answer fields, but the instructor can add as many as they desire. The Questions are scrambled automatically when displayed to the student. Even though they are not displayed in the same position, the Answer for Question 1 still matches with Question 1, etc.

    The Questions are displayed to the student as a numbered list (in random order) with corresponding drop-down menus for selecting the appropriate Answers. The first Answer menu matches to whichever Question is the first in the displayed list (not necessarily the first Question set by you). Click Save to add the problem to the pool.
  • question_answerHow to Create a Parson's Problem?

    Parson's problems are puzzles that require students to arrange given lines of code in the correct order. To create one, follow the steps to create a Problem and select Parson's Problem. With the exception of the answer fields, follow guidelines in How do I create a Problem.

    There are several fields specific to the Parson's problems. First, the instructor has to select the programming language (currently Java and Python are supported). Java is the default one. The choice for Code Indentation determines whether the student solution will be checked for a correct indentation.

    In the Model Solution area the instructor has to enter a correct solution of the problem - the lines and blocks of code which will be given to the student to arrange. To combine two consecutive code lines in a block, which will be dragged by the student as one unit, type ' ##' at the end of the first line. You have to combine in blocks code lines, such as assignment statements and closing brackets ('}'), which can be arranged correctly in different ways. Otherwise the system may mark a correct student solution as wrong.

    In addition to the correct code lines for solving the problem, the instructor may include ‘distractors’. Distractors are code lines which do not belong to the correct solution and should not be dragged to the solution area. A distractor is indicated by typing '#distractor' at the end of the code line, e.g. ' i--; #distractor'. One key difference in the typical problem setup is that Difficulty serves a greater purpose here. The system automatically selects how many of the provided distractors should be shown to the student based on the specified level of problem difficulty. As Difficulty increases, the more distractors will be shown to the students.

    Fill in the required information and click Save to add the problem to the pool.

  • question_answerWhat are Dynamic problems?
    Dynamic problems are problems created with Lua code that have far more versatility than static problems. Instructors can create dynamic problems which have random elements to questions. In some cases, they might also create static questions with dynamic grading when there are many different correct answers possible. This includes support for using external tools such as compilers and interpreters as part of the grading process so that you can grade simple programming problems even when there are many possible variations on the solution. There are two types of dynamic problems: Templates and Raw Lua. A Template problem allows instructors to enter the formatted text of a problem and only fill in specific details dynamically using values set up in the code portion. A Raw Lua problem is more flexible than Template, but requires more programming in Lua. Instructors should use Raw Lua problems only if they are trying build a problem where the text or structure is likely to vary radically between the different randomized instances. We expect that this will be quite rare. Most things can be done with Template problems.
  • question_answerHow to Create a Dynamic (Template) Problem?

    Templated dynamic questions allow instructors to write questions and evaluate answers programmatically using Lua code. To create one, go to the Course Content menu and select Problems. Click Create Problem and select Dynamic (Template). The first part of the problem setup has standard configuration options of a Problem (Problem Preview, Skills, etc.). Click the blue Help button with a “?” mark on the top right-hand side of the Problem Info to show help for Template problems.

    The four distinct sections require more nuance: Add Dependency, Dependent Libraries, Set-Up Code, and Template. Instructors can add any Lua Libraries that they have uploaded as dependencies by clicking the Add Dependency drop-down menu and selecting their desired library. The new dependency will appear in the Dependant Libraries section. Lua code is used to set up any needed variables and perform any computations needed including computing the answer that the student answer will be checked against. It should be included in the Set-Up Code section. The problem itself should be below.

    To display a variable in the Problem Area, it must be enclosed in pipes (vertical lines) enclosed in brackets (ex. [|x|]). To include Lua code, it must be enclosed in curly braces enclosed in brackets (ex. [{Some code for your thoughts}]). A piece of code that must be included at least in the problem is the make_answer function which is used to create answer blanks. If this is omitted, students will not be able to answer the generated question. More than one answer blank can be included in a single problem. The make_answer function takes five arguments: The name of the answer blank (displayed to students when describing their results), the type of answer blank, the size, the answer checking function to be used (several of which are provided), and the maximum amount of points the student can receive for the answer blank. A simple example make_answer is provided when creating a Template problem in the Template section.

    Instructors can create multi-part problems, where the next part is dependant on the previous one. Simply click the Add Part button to add an additional Template section. Instructors can also test their problems before creating them in by clicking the Try Out Question button at the bottom of the page. The Random Seed field affects any random variables that the Instructor might have included. When the same Random Seed is used, the same problem will be generated. This can be used to debug problems if students see a version of the problem with an error and to check that variations of the problem are correctly being generated and graded. Click Save to add the problem to the problem pool.

  • question_answerHow to Create a Dynamic (Raw Lua) Problem?

    Dynamic problems allow instructors to write problems and evaluate answers programmatically using Lua code. To create one, go to the Course Content menu and select Problems. Click Create Problem and select Dynamic (Raw Lua). The first part of the problem setup has standard configuration options of a Problem (Problem Preview, Skills, etc.). Click the blue Help button with a “?” mark on the top right-hand side of the Problem Info to show the documentation for Raw Lua problems.

    It should be noted that Templated problems and Raw Lua have practically the same capabilities. We expect that instructors will usually only use Raw Lua problems when they are building a very complex problem where the structure of the problem instances might vary wildly so that they can't write a single template. Anything done in Raw Lua can generally be done in Templated problems, and this is the recommended approach.

    There are four sections that require attention: Add Dependency, Dependent Libraries, Number of Parts, and Code. Both Add Dependency and Dependent Libraries function exactly as they do with Templated problems, allowing Instructors to add any Lua libraries they have uploaded. The Number of Parts is a function similar to the Add Part button in Templated problems. If you wish for a multi-part problem, it must be manually specified by entering the amount of parts in the field. The Code section is where the problem is made using Lua Code. The code, when run, should define the following: part_1_text, evaluate_answer_1, and part_1_max_points.

    The part_1_text function takes no input and produces a string which represents the HTML for the first problem. If you have more than one part to this problem, you should also define similar functions call part_2_text, part_3_text, etc. The evaluate_answer_1 function takes in a table which maps the names of the form inputs from part 1 to values which are the answers provided for those form inputs by the user. It returns a table which maps the names of the form inputs to a table containing two values: “success”, which maps to a boolean indicating whether or not the answer was sufficient to continue to the next part and "value", which maps to a floating point number indicating how many points were awarded for that answer. The part_1_max_points function takes no input, but returns a table which maps the name of each input to the maximum possible number of points a student could earn for that problem. An example of proper Raw Lua code is in the Code section when creating the problem.

    As with Template problems, Instructors can test their problems before creating them by pressing the Try Out Question button at the bottom of the page. The Random Seed field affects any random variables that the Instructor might have included. Click Save to add the problem to the problem pool.

  • question_answerWhat is the Student View and the Test Student?

  • question_answerWhat is OneUp Chat and How It Works?

    OneUp Chat is an online chatroom for OneUp that allows Instructors and Students of a course to communicate outside of the classroom. Though it’s primary use is to allow students to ask Instructors questions, it has great versatility. Instructors can create live discussion channels, have students post questions to different channels to be answered during class, and create channels for different groups for group projects.

    To Enable the Chat feature in OneUp, in the Course Information menu, select Course Configuration. Then, navigate to the option to Set Chat. Click Enable Chat Support and then click Save at the bottom of the page. The Chat option will appear under the Game Features menu.

    After enabling the Chat tool, you can use it by selecting Chat in the Course Tools menu. To Create a Channel, press the ‘+’ button next to the Create Channel label. A pop-up will appear with the options to set the name of the channel, the topic, and inclusivity of the channel. Click Create to add the Channel. Currently, only Instructors can create channels for students to participate in. Different channels are denoted by the ‘#’ symbol and can be switched between freely.

    Note that students are anonymous to each other in the Chat. Only the Instructor can see student names in all channels. The students are presented with their Avatars.

    You can use Emojis in your messages! Be cool and hip with the kids! Yo! Yo!

This is perhaps one of the most useful summary pages for the instructors. The link to the page is in the Course Reports menu.

  • question_answerSerious/Warm-Up Challenges Visualization

  • question_answerHow is Gamification Implemented and What are Rules?

    The OneUp platform includes a game engine which is responsible for executing Rules defined by the instructor. The Rules specify under what Conditions (based on challenges, activities and skills) specific game elements will be applied. The Rules have the format IF condition THEN action , for example, IF a student completes more than 20 challenges THEN award her a badge.

    After a student's interaction with OneUp, the game engine checks the rules to see if specific conditions are met. If they are, then the corresponding actions specified by the Instructor occur. These actions can vary: a Badge is awarded, Virtual Currency is awarded, a new Challenge is unlocked, etc. While sounding complex and intimidating, creating Rule Conditions can be simply thought of as specifying a Boolean expression.
  • question_answerHow do I Create a Rule?

    If you are creating a Badge, setting Virtual Currency Earning Rules, or configuring Progressive Unlocking, you will need to specify the Rule Conditions. The Rule Conditions section is always found at the bottom of the page. A simple Rule Condition (Atom Condition) consists of two parts, which will be compared. For example, a Condition would read like this: The percent of student score for Challenge X is more than or equal to 75.


    The Rule Conditions section is a drag-and-drop interface: drag a condition type (Atom, OR, or Forall) into the dotted rectangle to begin modifying it. Note the small AND followed by another dotted rectangle. This is for when you want multiple conditions to be true in order for the rule to fire (i.e. x = 2 AND y = 3). There are three condition types: Atom, OR, and Forall.


    The Atom condition is a single conditional statement (i.e. if (x = 5) would only be true if x = 5). Atom is the primary condition type, for they can apply to most situations. By default, an Atom condition type is the first condition in the Rule Conditions section. The Atom condition has three parameters: the Variable, the Operator, and the Threshold. The Variable is the specific condition that is being checked (i.e. student percentage score on a challenge). The Operator can be one of five options: greater than (>), less than (<), equal to (=), greater than or equal to (>=), and less than or equal to (<=). The operators function just as they do mathematically. The Threshold simply specifies the value to which the Variable will be compared. By default, the Threshold is a number, but if the Value checkbox is unchecked, the Instructor can choose another variable to compare to the first.

    The OR condition specifies that when one of two or more conditions are true then the entire condition would be true (i.e. the statement if (x < 2) OR (x> 4) would be true if x = 1 or 5). OR is useful if the Instructor desires multiple ways to satisfy a condition.


    The Forall condition allows Instructors to specify how broad the rule is. There are three selectors in Forall: For, Of the following, and Items to add. The 'Of the following' selector allows the Instructor to select the objects to be considered for the condition: Challenges, Activities, Topics, and Categories (Activity Categories). The 'Items to add' selector allows the Instructor to add one or more items of the type specified in the 'Of the following' selector. The checkbox next to the 'Items to add' selector will select all of the items of the selected type if checked (i.e All Activities). The For selector simply specifies whether each or all of the specified selections shall be considered. At the bottom of the Forall condition is included a dotted rectangle. You have to drag an Atom or OR condition there. The dragged Atom or OR condition will work just the same, but with the added restrictions set by the Instructor. Once an Atom condition is placed, the available variables in the drop-down menu become more specific depending on the item type selected in the 'Of the following' selector.


    To remove a condition from the area, drag it to the Trash box at the upper left of the section.
